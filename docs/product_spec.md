# Product Spec: ResearchPilot

## 1. 背景

研究生课程、论文开题和技术选型都需要快速完成学术调研。传统流程依赖人工检索、阅读、整理和写作，存在耗时长、证据来源分散、引用追踪困难和报告质量不稳定等问题。

ResearchPilot 目标是构建一个 Agentic AI 原生学术调研系统，把“检索论文、理解论文、组织证据、生成报告、评估质量”封装成可观测、可复现的多 Agent 工作流。

## 2. 用户目标

- 输入研究主题后自动获得结构化研究报告。
- 查看每一步 Agent 执行轨迹。
- 查看论文来源、摘要、相关性和 Agent 总结。
- 基于已构建知识库继续追问。
- 在答辩或演示时清晰展示真实 API 调用状态、失败原因和 Agent 执行日志。

## 3. 功能范围

### MVP

- 创建研究任务。
- Planner Agent 生成关键词、研究问题和报告大纲。
- Search Agent 通过 OpenAlex / arXiv 检索论文。
- Reader Agent 总结论文。
- RAG Store 写入本地知识库。
- Writer Agent 生成 Markdown 报告。
- Evaluator Agent 评估报告质量。
- Next.js 前端展示任务、论文、报告、评分和问答。

### 非目标

- 用户登录和权限管理。
- PDF 全文解析。
- 多用户协作。
- 复杂论文爬虫。
- 生产级推荐系统。

## 4. 成功指标

- 单个真实任务可以完整输出 Agent 轨迹、论文结果、报告和评估。
- API Key 缺失时任务进入 failed 状态，并在前端展示明确错误信息。
- 每个任务至少输出 3 篇论文、1 份 Markdown 报告和 1 组评估分数。
- 所有 Agent 步骤都有可观测日志。

## 5. 风险处理

| 风险 | 处理方式 |
| --- | --- |
| LLM API 不稳定 | 前端展示 failed 状态，提示检查 API Key、网络或模型服务 |
| 论文 API 请求失败 | 前端展示 failed 状态，提示检查 OpenAlex/arXiv 网络访问 |
| Chroma 安装失败 | JSON 词法检索回退 |
| 前端联调慢 | API 结构稳定，页面轮询 |
